# 🐱 云养猫模拟器 · 任务规划

## 📌 最终目标

做一个**全栈养猫小游戏**。用户执行 `uv run main.py` 后，浏览器自动打开，就能看到一只可爱的小猫在屏幕上活动，用户可以喂食、撸猫、逗猫、哄睡、拍照。小猫有 4 个维度的状态（饱食度、心情、精力、亲密度），会随时间变化，就算关掉浏览器，后端也在持续运行，小猫依然在"生活"。

---

## 🛠️ 技术栈（必须遵守）

- **后端**：Python + FastAPI + WebSocket
- **前端**：HTML + Canvas 或 SVG + 原生 JavaScript（**不要用 React/Vue/任何框架**）
- **数据库**：SQLite
- **依赖管理**：`uv` 的 PEP 723 inline script metadata
- **项目结构**：单文件 `main.py`，HTML/CSS/JS 全部以字符串内嵌

---

## 📋 执行步骤

请严格按以下 6 步顺序执行。每完成一步，先在脑中验证这一步的产出符合要求，再进入下一步。

---

### Step 1 · 搭建项目骨架（约占 10% 工作量）

**要做的事**：
1. 创建 `main.py` 文件
2. 在文件顶部用 PEP 723 格式声明依赖（fastapi、uvicorn、websockets 等）
3. 初始化 FastAPI app
4. 编写 `if __name__ == "__main__"` 入口，使用 uvicorn 启动，端口 8000
5. 加入自动打开浏览器的逻辑（`webbrowser.open`）
6. 先创建一个空的 `/` 路由，返回一句 "Hello Cat" 测试

**完成标准**：执行 `uv run main.py` 能启动服务器且浏览器自动打开，看到 "Hello Cat"。

---

### Step 2 · 设计小猫的数据模型和状态机（约占 15%）

**要做的事**：
1. 定义 `Cat` 数据类，包含字段：
   - `name`（小猫名字，默认"咪咪"）
   - `hunger`（饱食度 0-100，初始 80）
   - `mood`（心情 0-100，初始 70）
   - `energy`（精力 0-100，初始 90）
   - `bond`（亲密度 0-100，初始 0）
   - `current_state`（当前状态枚举）
   - `last_update`（上次更新时间戳）
2. 设计 5 种状态的判定逻辑：
   - `sleeping`：精力 < 20
   - `hungry`：饱食度 < 30
   - `grumpy`：心情 < 25
   - `happy`：心情 > 70 且 饱食度 > 50
   - `normal`：其他
3. 初始化 SQLite 数据库，创建 `cats` 表存储小猫状态
4. 编写 `load_cat()` 和 `save_cat()` 函数

**完成标准**：重启服务器后小猫状态不丢失（从 SQLite 读取）。

---

### Step 3 · 实现状态衰减后台任务（约占 15%）

**要做的事**：
1. 使用 FastAPI 的 `lifespan` 事件启动一个 `asyncio` 后台任务
2. 后台任务每 10 秒执行一次：
   - 根据经过时间计算各属性应衰减多少（饱食度每 30 秒 -1，心情每 60 秒 -1）
   - 更新小猫状态到 SQLite
   - 重新判定当前状态（睡觉/饥饿/开心等）
3. 设置 WebSocket 连接管理器，能广播消息给所有客户端
4. 每次状态更新后通过 WebSocket 推送最新数据

**完成标准**：即使没有任何前端操作，后端日志也能看到小猫状态在持续变化。

---

### Step 4 · 实现 5 种互动 API（约占 20%）

**要做的事**：
1. `POST /api/feed` — 喂食接口
   - 接收参数 `food_type`：`kibble`（普通猫粮）、`fish`（小鱼干）、`veggie`（蔬菜）
   - 三种食物对 hunger、mood 影响不同
2. `POST /api/pet` — 撸猫接口
   - 接收参数 `body_part`：`head`、`chin`、`belly`、`tail`
   - 不同部位效果不同：头顶和下巴舒服、肚子警戒、尾巴生气
3. `POST /api/play` — 逗猫接口
   - 扣 5 点 energy，加 8 点 mood，加 2 点 bond
   - 如果 energy < 20 返回"小猫累了"
4. `POST /api/sleep` — 哄睡接口
   - 进入睡觉状态，每秒 energy +2 直到满
5. `POST /api/photo` — 拍照接口
   - 记录当前状态到 `photos/` 目录（保存 JSON 快照即可，真正的图片由前端生成）
6. `GET /api/cat` — 获取小猫当前状态
7. `WS /ws` — WebSocket 端点

**完成标准**：用 curl 或 Postman 测试每个接口都能正常改变小猫状态。

---

### Step 5 · 编写前端页面（约占 30%）

**要做的事**：

**5.1 HTML 骨架**
- 顶部：小猫名字 + 亲密度爱心
- 中部：三个进度条（饱食度、心情、精力）
- 主画面：Canvas 或 SVG 区域展示小猫
- 对话气泡：显示小猫当前说的话
- 底部：5 个互动按钮

**5.2 小猫绘制（关键！）**
- 用 Canvas 2D 或 SVG 画一只小猫
- 必须有动画：
  - **呼吸起伏**：idle 时身体轻微上下浮动
  - **尾巴摇摆**：持续左右摆动
  - **眨眼**：每 3-5 秒眨一次
  - **走路**：喂食或逗猫时小猫会移动
  - **表情切换**：5 种状态下小猫表情不同

**5.3 互动逻辑**
- 5 个按钮分别调用对应 API
- 喂食按钮弹出三选一菜单（猫粮/小鱼干/蔬菜）
- 撸猫：鼠标按住拖动，检测鼠标在小猫哪个部位
- 逗猫：鼠标移动显示逗猫棒图标，小猫追鼠标
- 哄睡：画面变暗，小猫睡觉动画

**5.4 视觉效果**
- 撸舒服时周围飘爱心粒子
- 吃东西时食物从上方掉落动画
- 逗猫时小猫追逐动画（蹲伏 → 扭屁股 → 扑）

**5.5 对话系统**
- 至少准备 15 句台词，根据状态随机显示
- 饥饿时："喵呜...我想吃小鱼干"
- 开心时："呼噜呼噜~"
- 睡觉时："Zzz..."
- 被撸时："咕噜咕噜咕噜"
- 被摸肚子时："嘶!别碰那里!"
- 闹脾气："哼!不理你了"

**5.6 WebSocket 连接**
- 页面加载时连接 `/ws`
- 收到状态更新时刷新进度条、表情、气泡
- 断线自动重连

**完成标准**：浏览器打开能看到一只活灵活现的小猫，点按钮有反馈。

### Step 6 · 打磨细节（约占 10%）

6. 处理边界情况：数值不能超过 100 或低于 0

**完成标准**：整体体验流畅,没有明显 bug,视觉和谐。
