# 论文工作台搭建

这个技能只做一件事：把混在一起的论文材料整理成一个可重复使用的项目工作台，并给每类材料写清楚使用规则。

它不是写论文技能，也不是找文献技能。它的目标是让后续人工智能操作先分清材料身份，避免把文献、草稿、导师意见、会议纪要、实验数据混着用。

## 适用场景

当用户有以下材料时使用：

- 论文初稿、章节草稿、摘要或提纲
- PDF 文献、阅读笔记或参考文献列表
- 导师意见、组会纪要、合作者反馈或审稿意见
- 数据表、实验结果、图表草稿
- 用户想把这些材料整理成一个能长期使用的论文项目目录

## 默认输出

在用户指定的项目目录下创建这些工作区：

```text
01_文献/
02_论文草稿/
03_反馈材料/
04_中间文本/
05_输出报告/
项目总说明.md
```

每个工作区都要生成一个 `说明.md`。

如果用户已有类似目录，不要强行重建。优先沿用现有结构，并补齐缺失的说明文件。

## 工作区职责

### 01_文献

放可引用资料，包括 PDF 论文、学位论文、综述、出版页面导出文本、阅读笔记和参考文献列表。

规则要写明：

- 文献可以作为证据来源，但必须检查原文是否支撑正文句子。
- 总结文献时要提取研究问题、方法、核心结论、关键证据、局限性和可支撑的论文论点。
- 不要因为标题相关就判定可以引用。
- 每条证据都要保留来源位置，例如页码、章节、标题、行号或原文片段。

### 02_论文草稿

放作者的论文正文材料，包括初稿、章节草稿、摘要、引言、相关工作、方法、实验和结论。

规则要写明：

- 草稿是作者材料，不等于已证实事实。
- 不要编造实验结果、公式、图表、参考文献或数据。
- 遇到“待补充”“估算值”“需检索”等内容，要标为作者待确认。
- 引用建议必须回到文献区找证据。

### 03_反馈材料

放导师意见、组会纪要、合作者意见、审稿意见和修改要求。

规则要写明：

- 反馈材料只能作为修改任务来源，不能当作文献证据。
- 先提取行动项，再区分紧急问题、结构问题、证据问题、实验问题和格式问题。
- 涉及补文献的反馈，应交给文献证据匹配流程。
- 涉及补实验、补公式、补图表的反馈，应标为作者待补充。

### 04_中间文本

放从 Word、PDF 或网页中提取出来的可读文本。

规则要写明：

- 原始 Word 和 PDF 不覆盖、不改动。
- 提取文本尽量保留标题、章节、页码或段落线索。
- 后续证据匹配优先读取中间文本，必要时再回看原文件。
- 如果提取质量差，要记录问题，例如乱码、缺页、扫描版无法识别。

### 05_输出报告

放分析结果和检查报告。

常见报告包括：

- 论点-证据匹配表
- 引用补充建议
- 审稿人模拟检查报告
- 导师意见行动项清单
- 下一稿修改清单

规则要写明：

- 报告必须区分已确认事实和需要作者确认的内容。
- 引用建议必须有原文证据。
- 证据不足的句子要明确标注，不要硬凑引用。

## 项目总说明

`项目总说明.md` 应包含：

- 项目名称或论文题目
- 当前目标
- 现有材料清单
- 工作区说明
- 第一轮建议流程
- 硬性规则

硬性规则默认包括：

1. 不编造参考文献。
2. 不编造实验结果。
3. 不把导师意见、会议纪要或审稿意见当作文献证据。
4. 每条引用建议必须能回到原文证据。
5. 如果文献只能背景相关，必须标为背景支撑。
6. 审查类流程默认只生成报告，不直接改正文。

## 操作流程

1. 扫描用户给出的目录，识别 Word、PDF、Markdown、文本、表格和图片文件。
2. 判断现有材料类型：文献、草稿、反馈、数据、输出。
3. 创建缺失的工作区。
4. 为每个工作区写 `说明.md`。
5. 写 `项目总说明.md`。
6. 不移动原始文件，除非用户明确要求整理归档。
7. 最后报告生成了哪些目录和文件。

## 本地工具调用

执行时优先使用本地文件工具，不依赖特定智能体平台。

### 1. 扫描目录

优先用：

```bash
find . -maxdepth 3 -type f | sort
```

如果需要看文件大小：

```bash
find . -maxdepth 3 -type f -printf '%p\t%s bytes\n' | sort
```

### 2. 判断文件类型

按扩展名初步分类：

- `.pdf`：放入文献区候选。
- `.docx`、`.doc`、`.md`、`.txt`：根据文件名和内容判断是草稿、反馈还是笔记。
- `.xlsx`、`.csv`：放入数据材料候选。
- `.png`、`.jpg`、`.svg`、`.pdf` 图件：放入图表或输出候选。

### 3. 创建工作区

使用等价的目录创建工具生成：

```bash
mkdir -p 01_文献 02_论文草稿 03_反馈材料 04_中间文本 05_输出报告
```

### 4. 写入说明文件

每个工作区写一个 `说明.md`。如果文件已存在，不要覆盖用户内容；应先读取，再只补充缺失规则。

### 5. 输出清单

完成后运行：

```bash
find . -maxdepth 2 -type f | sort
```

把新生成的文件清单报告给用户。

## 禁止事项

- 不要在这个技能中改写论文正文。
- 不要在这个技能中判断引用是否成立。
- 不要删除或覆盖用户原始文件。
- 不要把所有规则堆进一个巨大的总说明；每个工作区只写自己的规则。
