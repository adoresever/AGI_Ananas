# 审稿人模拟检查

这个技能像严厉审稿人一样检查论文草稿，但默认不直接改正文。

它的目标是发现投稿风险：证据不足、引用不成立、实验缺失、方法说不清、结论过强、结构不闭合，以及导师或审稿人已经指出但尚未解决的问题。

## 适用场景

当用户想做以下事情时使用：

- 投稿前检查论文风险
- 按审稿人视角挑问题
- 根据导师意见或会议纪要生成修改优先级
- 检查草稿中哪些内容不能提交
- 检查证据、实验、公式、图表、引用和结论是否闭合

## 输入

可接受材料：

- 论文初稿、章节草稿、摘要或提纲
- 导师意见、组会纪要、合作者意见
- 审稿意见或编辑意见
- 文献证据匹配报告
- 目标期刊要求
- 实验结果表、图表清单、公式草稿

如果没有草稿，只能做修改任务整理，不能模拟完整审稿。

## 默认立场

- 只生成报告，不直接改正文，除非用户明确要求改写。
- 不编造实验结果、公式、图表、文献或数据。
- 把“作者待补充”和“可由文字修改解决”的问题分开。
- 先指出高风险问题，再指出小问题。
- 如果草稿里有估算数据、待补充占位、缺图、缺表、缺公式，要明确标出。
- 如果引用证据不足，应引用文献证据匹配报告或要求先运行文献证据匹配。

## 本地工具调用

执行时优先读取已经生成的中间文本和报告，避免反复直接解析原文件。

### 1. 扫描可用材料

```bash
find . -maxdepth 3 -type f | sort
```

重点查看：

```text
04_中间文本/
05_输出报告/
导师意见.md
会议纪要.md
审稿意见.md
目标期刊要求.md
```

### 2. 读取草稿文本

如果已有：

```text
04_中间文本/论文初稿.extracted.md
```

优先读取它。没有中间文本时，再从 Word 或 Markdown 草稿提取。

### 3. 查找高风险占位

用全文搜索工具定位风险词：

```bash
rg -n "待补充|估算|预计|缺失|待插入|需检索|不完整|约|TODO|TBD" .
```

这些命中不是结论，只是审稿风险候选，需要读上下文后判断。

### 4. 查找引用和证据问题

如果有文献证据匹配报告，先读取：

```text
05_输出报告/文献证据匹配报告.md
```

若没有该报告，但草稿依赖大量引用，应建议先运行文献证据匹配技能。

### 5. 写入报告

默认写入：

```text
05_输出报告/审稿人模拟检查报告.md
```

多轮测试可写为：

```text
05_输出报告/审稿人模拟检查报告_第一轮.md
```

## 检查维度

### 1. 证据和引用

检查：

- 关键背景判断是否有文献支撑
- 引用是否能直接支撑正文
- 是否把背景文献当成直接证据
- 是否缺少反方文献或局限性文献
- 是否有不完整参考文献条目

### 2. 实验和数据

检查：

- 实验结果是否真实完整
- 是否存在估算值、占位符或待补充表格
- 主实验、消融实验、对比实验是否足够
- 指标、数据集、划分方式、基线方法是否说明清楚

### 3. 方法和公式

检查：

- 方法流程是否可复现
- 核心算法是否有形式化表达
- 公式、变量、训练目标是否完整
- 图表是否能帮助理解方法

### 4. 结构和论证

检查：

- 引言是否说明背景、缺口、问题和贡献
- 研究问题和结论是否逐一对应
- 相关工作是否覆盖最新和最相关研究
- 方法贡献和工程贡献是否混在一起
- 讨论和局限性是否和实验结果对应

### 5. 投稿风险

检查：

- 哪些问题会导致直接拒稿
- 哪些问题会导致大修
- 哪些是小修或格式问题
- 哪些必须由作者补实验或补材料

## 输出格式

默认输出到：

```text
05_输出报告/审稿人模拟检查报告.md
```

报告结构：

```text
# 审稿人模拟检查报告

## 输入材料

## 总体判断

## 高优先级问题
| 问题 | 位置 | 风险 | 建议 |
|---|---|---|---|

## 证据和引用问题

## 实验和方法问题

## 结构问题

## 需要作者确认的问题

## 下一稿修改优先级
```

## 风险等级

使用这些风险等级：

- `高`：会严重影响投稿可信度，例如实验缺失、数据为估算值、核心公式缺失、关键引用不成立。
- `中高`：会明显削弱论文说服力，例如相关工作覆盖不足、图表缺失、研究问题和结论不对应。
- `中`：需要修改，但通常不单独构成拒稿理由。
- `低`：格式、表达、局部清晰度问题。

## 与其他技能的关系

- 如果项目材料混乱，先用 `paper-workbench`。
- 如果引用和证据不清楚，先用 `literature-evidence-matcher`。
- 本技能负责审查风险和修改优先级，不负责直接补文献或改正文。

## 禁止事项

- 不要替作者编造实验结果。
- 不要把外部文献当作作者实验结果。
- 不要直接声称某项修改已完成，除非材料中确实存在修改证据。
- 不要把导师意见或会议纪要当成论文事实。
- 不要默认帮作者改正文；默认只出审稿报告。
